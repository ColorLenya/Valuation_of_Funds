## 2. 恒生 OAuth2 授权（启动时获取 Token）

**需求**：项目启动时自动向恒生开放平台申请 OAuth2 访问令牌，并打印返回结果。

**授权方式**：

- URL：`https://open.hscloud.cn/oauth2/oauth2/token`（POST）。
- Body：`application/x-www-form-urlencoded`，参数 `grant_type=client_credentials`。
- Headers：`Authorization: Basic Base64Encode(app_key:app_secret)`，UTF-8 编码。

**实现**：

- 配置类：`HscloudOAuthProperties`，绑定 `application.yml` 中的 `hscloud.oauth`（token-url、app-key、app-secret）。
- 启动时执行：`HscloudTokenRunner` 实现 `ApplicationRunner`，在 `run()` 中发送 POST 请求，成功则打印返回结果，失败则捕获并打印 401 响应体便于排查。

## 4. 恒生 OAuth2 授权失败：401 [no body]

**现象**：启动时请求 token 返回 401，且无响应体。

**代码调整**：

- 对 401 捕获 `HttpClientErrorException`，用 `getResponseBodyAsString()` 打印响应体；若无 body 则提示检查 App Key/Secret 及应用是否启用。
- 请求体改为明文字符串 `grant_type=client_credentials`，并设置 `Content-Type: application/x-www-form-urlencoded;charset=UTF-8`。
- 对 app_key、app_secret 做 `trim()`。

**排查建议**：核对 Key/Secret、应用是否启用、环境（测试/生产）及 IP 白名单等。

## 5. Postman 能获取 Token，代码 401（代码是否有问题）

**结论**：Postman 使用测试环境能成功，说明凭证正确，问题多为**请求的 URL 或请求格式**与 Postman 不一致。

**修改**：

1. **使用测试环境 Token 地址**  
   恒生测试环境与生产环境域名不同：  
   - 测试环境：`https://sandbox.hscloud.cn/oauth2/oauth2/token`  
   - 生产环境：`https://open.hscloud.cn/oauth2/oauth2/token`  
   将 `application.yml` 中默认 `token-url` 改为测试环境地址。

2. **请求与 Postman 对齐**  
   - Body 使用 `MultiValueMap` 构造表单，由 Spring 按 `application/x-www-form-urlencoded` 序列化。  
   - Content-Type 使用 `MediaType.APPLICATION_FORM_URLENCODED`（不额外带 charset）。  
   - 增加 `Accept: application/json`。  
   - 移除调试用的 `System.out.println(authHeader)`。

若 Postman 实际用的是 `open.hscloud.cn`，将配置中的 `token-url` 改回生产地址即可。

## 6. HscloudOAuthProperties 的作用

**作用**：作为 Spring Boot 的**配置属性类**，把配置文件里的恒生 OAuth2 配置集中映射成 Java 对象，供其他 Bean 注入使用。

- `@ConfigurationProperties(prefix = "hscloud.oauth")`：将 `application.yml` 中 `hscloud.oauth` 下的项绑定到该类字段。
- 对应关系：`token-url` → `tokenUrl`，`app-key` → `appKey`，`app-secret` → `appSecret`。
- `HscloudTokenRunner` 注入该类，调用 `getTokenUrl()`、`getAppKey()`、`getAppSecret()` 发起 token 请求；后续若有其他组件需要同一配置，也可直接注入此类。

## 7. tokenUrl 默认值是否会生效

**问题**：`HscloudOAuthProperties` 中字段默认值  
`private String tokenUrl = "https://sandbox11111.hscloud.cn/oauth2/oauth2/token";`  
是否会生效。

**结论**：

- **配置里没有 `token-url`**：Spring 不会调用 setter，字段保留 Java 中的默认值，**会生效**。
- **配置里有 `token-url`**（如在 `application.yml` 中已配置）：Spring 会调用 `setTokenUrl(...)` 将配置值写入，**会覆盖**字段默认值，此时以配置文件为准。

因此，若 yml 中已配置 `token-url`，改 Java 里的默认值不会改变运行时的值；要改地址应改 yml，或删除 yml 中的 `token-url` 以使用 Java 默认值。

## 8. HscloudTokenRunner 为什么在项目启动时执行

**原因**：该类实现了 Spring Boot 的 **`ApplicationRunner`** 接口。

- 任何被 Spring 管理的 Bean，只要实现 `ApplicationRunner`，在**应用完全启动后**（上下文已刷新、可接收请求前），Spring 会自动调用其 `run(ApplicationArguments args)`。
- `HscloudTokenRunner` 上有 `@Component`，会被扫描为 Bean，且实现了 `ApplicationRunner`，因此启动阶段会被调用。
- 执行时机：在 `SpringApplication.run(...)` 过程中，在“上下文就绪”与“应用开始对外服务”之间，Spring 会收集所有 `ApplicationRunner` 与 `CommandLineRunner` 的 Bean 并依次执行其 `run` 方法。

因此，项目一启动就会自动执行一次获取恒生 token 的逻辑。

## 9. 如何生成一个 Spring Boot 项目（模拟人工步骤）

### 9.1 事先想清楚

- Java 版本：一般 17 或 21（Spring Boot 3 至少 17）。
- 构建工具：Maven 或 Gradle。
- 项目存放目录和名称。

### 9.2 方式一：Spring Initializr 网页（最常用）

1. 打开 **https://start.spring.io**。
2. 选 **Project**：Maven；**Language**：Java；**Spring Boot**：稳定版本（如 3.2.x、3.3.x）。
3. 填 **Project Metadata**：Group（如 `com.example`）、Artifact（如 `my-demo`）、Name、Package、Packaging 选 Jar、Java 选 17 或 21。
4. 点 **ADD DEPENDENCIES**：至少勾选 **Spring Web**，按需加 JPA、MySQL Driver、Lombok 等。
5. 点 **GENERATE** 下载 zip，解压到目标目录。
6. 用 IDEA：File → Open 选解压后的目录（含 `pom.xml` 的层级）；Eclipse：Import → Maven → Existing Maven Projects。

### 9.3 方式二：IDEA 内置 Spring Initializr

1. File → New → Project，左侧选 **Spring Initializr**，SDK 选 JDK 17/21，Service URL 默认。
2. Next 后填 Group、Artifact、Name、Package、Type 选 Maven、Packaging 选 Jar、Java 选 17/21。
3. Next 进入依赖选择，勾选 Spring Web 等，再 Next。
4. 选 Project location，点 Create，IDEA 会创建并导入 Maven。

### 9.4 方式三：手写 Maven 项目结构（理解用）

1. 建目录：`项目名/pom.xml`、`src/main/java`、`src/main/resources`、`src/test/java`（包路径自定）。
2. 写 `pom.xml`：parent 为 `spring-boot-starter-parent`，依赖 `spring-boot-starter-web`、`spring-boot-starter-test`，build 中加 `spring-boot-maven-plugin`。
3. 在 `src/main/java` 下按包名建启动类，加 `@SpringBootApplication`，写 `main` 中 `SpringApplication.run(启动类.class, args)`。
4. 可选：在 `src/main/resources` 下建 `application.yml`（如 `server.port=8080`）。
5. 运行：IDEA 中运行主类，或命令行 `mvn spring-boot:run`。

### 9.5 生成后常见操作

- 先运行主类或 `mvn spring-boot:run` 确认能启动。
- 写一个 `@RestController` 的 `@GetMapping` 做健康检查或简单接口。
- 在 `application.yml` 中改端口、加数据库等配置。
- 按需在 `pom.xml` 中增加依赖并编写业务代码。

## 10. 获取 Token 后调用基金估值接口 + 前端页面

**需求**：拿到恒生 token 后，用 **Authorization: Bearer &lt;access_token&gt;** 调用基金估值接口；请求为 POST，Body 为 `secu_codes=xxx`；前端提供文本框输入证券代码、查询按钮，点击后发请求并展示返回结果。

**实现与各代码作用**：

| 文件 / 类 | 作用 |
|-----------|------|
| **HscloudTokenHolder** | 单例 Bean，内存中保存当前 `access_token`。Runner 写入，FundValuationService 读取，供调用业务 API 时拼成 `Bearer <token>`。 |
| **TokenResponse** | 恒生 `/token` 返回的 JSON 对应 DTO，字段：`access_token`、`token_type`、`expires_in`。用于 Jackson 反序列化，便于从响应里取出 `access_token`。 |
| **HscloudTokenRunner** | 在原有“请求 token 并打印”基础上：成功拿到响应后，用 `ObjectMapper` 将 JSON 转成 `TokenResponse`，取 `access_token` 并调用 `tokenHolder.setAccessToken(...)`，供后续基金估值请求使用。 |
| **HscloudApiProperties** | 配置属性类，绑定 `hscloud.api.fund-valuation-url`，即基金估值最新点位接口 URL（默认测试环境：`https://sandbox.hscloud.cn/gildatafund/v1/fundestimate/fundvaluation_lastpoint`）。 |
| **FundValuationService** | 从 `HscloudTokenHolder` 取 token，拼成 `Authorization: Bearer <token>`；用 RestTemplate 向 `fundValuationUrl` 发 POST，Content-Type 为 `application/x-www-form-urlencoded`，Body 为 `secu_codes=<传入值>`；返回恒生接口的原始响应字符串。 |
| **FundValuationController** | 提供 **POST /api/fund-valuation**，接收 JSON `{"secu_codes":"020156"}` 或 form 参数 `secu_codes`，调用 `FundValuationService.getFundValuationLastPoint(secuCodes)`，把结果原样返回；若未拿到 token 或调用异常，返回 503/500 及错误信息。 |
| **FundValuationRequest** | 请求体 DTO，仅字段 `secu_codes`（`@JsonProperty("secu_codes")`），供 Controller 接收前端 JSON 时反序列化。 |
| **application.yml** | 新增 `hscloud.api.fund-valuation-url`，指向恒生基金估值最新点位接口（测试/生产可在此切换）。 |
| **static/index.html** | 前端页：一个文本框（默认 020156）输入 `secu_codes`，一个「查询」按钮；点击后 `fetch` POST `/api/fund-valuation`，Body 为 `JSON.stringify({ secu_codes: 输入值 })`，在下方区域展示接口返回的原始内容。访问地址：`http://localhost:8080/` 或 `http://localhost:8080/index.html`。 |

**流程简述**：启动 → Runner 取 token 并写入 Holder → 用户打开页面、输入证券代码、点查询 → 前端请求 `/api/fund-valuation` → Controller 调 Service → Service 用 Bearer token + secu_codes 调恒生 → 结果返回前端并展示。

## 11. Token 响应解析失败：恒生返回中的 scope 等未知字段

**现象**：启动时日志报错，解析 token 响应失败，提示：`Unrecognized field "scope" (class com.valuation.funds.auth.TokenResponse), not marked as ignorable`。

**原因**：恒生 `/token` 接口返回的 JSON 里除了 `access_token`、`token_type`、`expires_in` 外，还有 `scope` 等字段；`TokenResponse` 只定义了上述三个属性，Jackson 默认遇到未知属性会抛错，导致反序列化失败，token 无法保存。

**修改**：在 `TokenResponse` 类上增加注解 **`@JsonIgnoreProperties(ignoreUnknown = true)`**。

**作用说明**：该注解告诉 Jackson，反序列化时忽略 JSON 中未在 DTO 中声明的字段（如 `scope` 及今后恒生新增的字段），只映射已定义的 `access_token`、`token_type`、`expires_in`。这样既不会因未知字段报错，又能正常取出并保存 `access_token`，后续基金估值请求可正常携带 Bearer token。
